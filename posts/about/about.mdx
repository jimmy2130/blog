## 程式學習經歷

### 啟蒙

從高中到大學，我有蠻多次接觸寫程式的經驗。條列如下：

- 高一的電腦課 (學 C++)
- 高一的前端網頁設計課程 (實驗性質居多，學 HTML、CSS)
- 大一的程式設計課 (學 C)
- 大一的數位邏輯實驗課 (學 Verilog)
- 師大通識課「運算思維與程式設計」(學 Scratch)
- 師大通識課「資料科學與程式設計」(學 Python)

真正讓我感受到突破的是「運算思維與程式設計」這堂通識課，該堂課使用視覺化的程式語言來教導程式設計。課程中，學生以拖拉磚塊的方式，在介面中組合出各種程式邏輯。

對我而言，比起學習程式的語法，培養解決問題的邏輯思考能力是比較重要的，從那堂課之後，我也開始對寫程式較為有信心。

### 學習網頁前端

2018 年寒假，我正式開始接觸網頁前端的技術，當時使用的教材是 HaHow 的「從入門到業界實戰 - UI / UX 前端網頁設計」課程，該課程的網頁實作部分，涵蓋了 HTML、CSS 和些微的 JQuery 技術。跟著那堂課的內容，我開始了解如何建構出簡單的網頁。另外，我也從那堂課中學會替元素加入邊框(border)的 CSS 偵錯法，利用邊框來檢查樣式有沒有被正確地套用在元素上。

<Sidenote title="不是百分之百有用">

這裡需要範例 Component。內元素的 Margin 會轉移到外元素，但如果外元素有 border，則無法轉移。

</Sidenote>

同年暑假，我在 Udemy 上買了 The Complete Web Developer: Zero to Mastery (以下簡
稱 ZTM 課程)。那堂課涵蓋了非常多的內容，除了 HTML、CSS 和 JavaScript 以外，最後
的大專案中，前端使用了 React、後端使用了 Node.js 與 Express.js、資料庫使用了
PostgreSQL，是一個包含了很多技術的網頁作品。

那堂課讓我了解到網頁開發是一件多麼不容易的事情，也是因為那堂課，我開始接觸JavaScript，學習用 JavaScript 來處理各種網頁互動的邏輯。

暑假結束後，升上大三的我，逐漸因為繁重的課業與專題研究，暫時停下學習網頁開發的腳步。

時間跳至 2021 年三月，此時的我正在就讀研究所碩一。我買了 Josh Comeau 的 CSS for JavaScript Developers 課程(以下簡稱 CSS for JS Dev)，來複習我已經生疏已久的 CSS技術。

那堂課和前兩堂課有許多不同之處。首先，它非常深入地介紹 CSS 的各種排版模式，以及它們的運作原理。再來，那堂課除了影片以外，還有文字解說 CSS 規則、大量的課後練習題，和一些小遊戲來幫助學生理解內容。每個章節最後都有一個專案，用來練習課程中學到的 CSS 技巧。

該堂課的大部分專案使用了 React 作為前端開發的框架，樣式的部分則是使用 styled-components 來撰寫。該課程的教學內容清晰易懂，對於我日後的網頁開發風格，也帶來了很深遠的影響。以下列舉幾個我學到的 CSS 觀念：

- Margin 塌陷 (Margin collapse)
- Z-index 與 Stacking context
- 各種排版模式 (Flow layout, Positioned layout, Flexbox, Grid layout)

在 CSS for JS Dev 課程中，有很多觀察 Figma 設計稿，並實作出對應網頁元件的練習。為了進一步練習此技巧，我加入了 Frontend Mentor 的會員，從該網站所提供的設計稿，練習實作網頁。實作過程中，除了要注意設計稿的各項細節以外，也必須兼顧網頁的可用性。我有將其中一部分的練習結果，放置在這個作品集網站裡。

完成 CSS for JS Dev 課程後，我買了 Kent C. Dodds 的 Epic React 來加強自己的 React 實力。那堂課是以一道道的程式碼練習題，加上影片解答的形式，來說明 React 的各種功能。當時的我，在 JavaScript 基礎不夠深厚的情況下，上那堂課變得非常吃力，常常感覺事倍功半，並沒有把 React 學得很好。

不過，課程最後的專案使用到了蠻多網頁開發的技術，如 React Router、React Query 等等，我後來有針對這些技術去做進一步的深入探索。和 Epic React 相比，React 官方所寫的 Beta 版技術文件，較為平易近人，該技術文件也讓我更了解如何使用 React。

### 學習演算法

ZTM 課程的作者，在 Udemy 上也有提供演算法相關的理論課與實作課。其中，理論課介紹了各種常見的資料結構，實作課則是解析了三十幾道的演算法問題。

有了基礎之後，2022 年寒假我買了 AlgoMonster 的課程，進一步練習各種演算法問題，包含雙指標(Two-pointers)、回溯法(Backtracking)、動態規劃(Dynamic Programming)等等。

另外，從 2020 年開始，每年十二月我都有參與 Advent of code 的活動，練習各式各樣的演算法問題。

<Sidenote title="Advent of Code">

Advent of Code 是由 Eric Wastl 舉辦的程式競賽，從 12 月 1 日到 12 月 25 日，每天都有兩道聖誕節主題的演算法問題。不限制使用何種語言作答，但是只有正確答出第一道問題，才能得知第二道問題的內容。

該網站的題目包羅萬象，除了測驗邏輯思考能力以外，有時候也需要數學的協助，才能得到正確解答，是一個相當有趣，又可以從中學到很多問題解決技巧的程式撰寫活動。

</Sidenote>

## 工作經歷

2019 年大三升大四的暑假，我到了鼎堅航太公司實習，主要負責的項目為自動化填報表，以及前端網頁開發。

所謂的自動化填報表，是指將大量的資料填入 Word 檔和 Excel 檔中。當時使用的技術為PHP Word 和 PHP Excel。事實上，比起自動帶入資料，更困難的是驗證資料的正確性，經由多次的嘗試，我建立起一套機制來檢查資料是否正確。確認完畢後，才將資料填入報表中。

<Sidenote title="現在還推薦 PHP Word 和 PHP Excel 嗎？">

將大量資料帶入 Word 檔或 Excel 檔，其實是文書處理中很重要的技能。舉凡製作獎狀、證書、名條等，都需要用到此技術。Word 軟體其實有內建這樣子的功能，因此不一定要使用 PHP Word 來處理資料帶入。

</Sidenote>

至於前端網頁開發，實作的項目為一個動態表單，該表單從介面設計到原型實作皆由我獨自完成。動態表單和傳統的待辦事項類似，但是增加了更多複雜的功能。當時使用的前端框架為 Vue，開發了一個月後，因為要申請研究所，因此後來將專案交給其他人繼續執行。

2019 年末，我離開了公司，準備提前進入研究所就讀。2020 年碩零期間，我決定嘗試運用公司學到的技術，來開發自動將五線譜轉換成簡譜的程式。

<Sidenote title="前情提要">

小時候學過音樂的我，斷斷續續有在廷廷的鋼琴窩上販賣琴譜。一直到離開鼎堅航太公司之後，我才開始投入較多的時間在琴譜製作上。

根據網站的規定，將做好的五線譜轉換成簡譜，審核通過後可得到新台幣 100 元。然而，五線譜轉換成簡譜的過程相當複雜，一直以來，我鮮少將做好的五線譜轉換成簡譜。

</Sidenote>

由於本人習慣使用 Mac 在 MuseScore 2 上製作琴譜，因此自動化程式也要能在上面執行。經由一系列的嘗試，最後採用的做法如下：

1. 用 JavaScript 解析琴譜的 musicxml 檔
2. 將解析的結果轉換成 AppleScript
3. 執行 AppleScript，將五線譜轉換成無數字版的簡譜
4. 使用自己寫好的 MuseScore 插件，填入音高

儘管該自動化程式還有不完美之處，它的功能已達到原先所設定的目標。穩定製作琴譜的兩年多來，這支程式成功為我轉換出約 700 份簡譜。
